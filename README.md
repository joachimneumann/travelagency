- a CUE model as the source of truth
- generated transport and runtime contract artifacts
- a Node backend
- a static HTML/ES-module frontend
- a reduced SwiftUI iOS shell
- local and staging deployment scripts

## Current Runtime Status

- Model and transport contract:
  - `model/`
  - generated by `tools/generator/generate_mobile_contract_artifacts.rb`
- Backend:
  - `backend/app/`
  - Node HTTP server with JSON/file persistence, Keycloak auth, pricing, PDF generation, and webhook integrations
- Frontend:
  - `frontend/pages/`
  - `frontend/scripts/`
  - static HTML plus browser ES modules
- Mobile:
  - `mobile/iOS/`
  - currently reduced to bootstrap, Keycloak login, session restoration, and a minimal authenticated shell
- Deployment:
  - `docker-compose.staging.yml`
  - `deploy/Caddyfile`
  - `scripts/staging/deploy_staging_all.sh`

## Common Commands

Generate contract artifacts:

```bash
ruby tools/generator/generate_mobile_contract_artifacts.rb
```

Start the local stack:

```bash
./scripts/local/deploy_local_all.sh
```

Start backend only:

```bash
./scripts/local/deploy_local_backend.sh
```

Wipe all booking data locally:

```bash
./scripts/content/wipe_local_bookings.sh --yes
```

Update staging:

```bash
./scripts/staging/deploy_staging_backend.sh
./scripts/staging/deploy_staging_frontend.sh
./scripts/staging/deploy_staging_backend_frontend.sh
./scripts/staging/deploy_staging_all.sh
./scripts/content/wipe_staging_bookings.sh --yes
```
